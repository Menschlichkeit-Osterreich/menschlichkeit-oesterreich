# Integrations-Matrix - Kernstack Priorisierung

**Stand:** 2026-04-20
**Kanonischer Scope:** aktive Pfade unter `apps/`, `automation/`, `mcp-servers/`, Repo-Root-Skripte.
**Nicht als produktive Wahrheit verwendet:** Legacy-/Mirror-Pfade wie `api.menschlichkeit-oesterreich.at/`, `crm.menschlichkeit-oesterreich.at/`, `web/`, `new/`.

## Ziel

Diese Matrix priorisiert reale Repo-Integrationen fuer ein kleines Team in drei Klassen:

- `notwendig`: fuer den stabilen Gesamtbetrieb des Repos
- `bereichsabhaengig`: nur fuer bestimmte aktive Teilbereiche erforderlich
- `optional`: nuetzlich, aber kein Blocker fuer den Kernbetrieb

## Priorisierter Kernstack (Empfehlung fuer kleines Team)

1. Bitwarden/BSM als einzige Secret-Wahrheit stabilisieren (`bsm-fetch-env`, `verify-payment-secret-wiring`).
2. Lokalen Kernbetrieb auf Docker + Postgres + API + Website + Stripe + Slack + SMTP absichern.
3. Erst danach bereichsabhaengige Stacks (CRM, n8n, Forum) sequenziell haerten.

## Core Runtime Secret Contract (Kernstack Pflichtwerte)

- API Start / Security: `DATABASE_URL`, `JWT_SECRET_KEY`, `ENVIRONMENT`
- Payment (Stripe): `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`
- Alerting: `ALERTS_SLACK_WEBHOOK`
- Mailzustellung (SMTP): `MAIL_USERNAME`, `MAIL_PASSWORD`, `MAIL_HOST`, `MAIL_PORT`, `MAIL_ENCRYPTION`, `MAIL_FROM_ADDRESS`, `MAIL_FROM_NAME`, `MAIL_REPLY_TO_ADDRESS`
- Regel: Jeder Pflichtwert hat genau eine kanonische BSM-Quelle in `secrets.manifest.json`.

## Stripe in VS Code (lokaler Best-Practice-Flow)

1. Empfohlene Extensions installieren (`VSCode: Install Recommended Extensions`).
2. Compound Launch `Debug: API + Stripe Webhooks` starten.
3. Fuer gezielte Repros alternativ Tasks nutzen:
   - `Stripe: Listen Payment Webhooks`
   - `Stripe: Trigger payment_failed`
4. Erwartung fuer lokalen Smoke-Test:
   - Webhook auf `http://localhost:8001/api/webhooks/stripe` antwortet mit HTTP 200.
   - Alerts werden ueber die aktive Notification-Chain (Slack/Mail) ausgeloest.

## Trennregeln zur Vermeidung von Tool-Chaos

- Keine zweite Secret-Wahrheit neben BSM (kein manuelles Nachpflegen produktiver Werte in mehreren Dateien).
- Kein Legacy-/Mirror-Pfad als Source of Truth fuer aktive Integrationen.
- Plesk ist Produktionsanker, aber kein Pflichtblocker fuer jeden lokalen Test.
- Codacy und weitere Zusatzintegrationen erst nach stabilem Kernstack priorisieren.
